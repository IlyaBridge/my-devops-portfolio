## О себе

DevOps Engineer с инженерным бэкграундом и практическим опытом эксплуатации и автоматизации распределённых систем.  
Имею опыт работы с облачной инфраструктурой, CI/CD пайплайнами, Infrastructure as Code и системами мониторинга и логирования.

В текущей профессиональной деятельности занимаюсь полным циклом развёртывания open-source платформы Open5GS для тестовых LTE/NR сетей: от инфраструктуры (ВМ на KVM/QEMU под компоненты Core Network) и сетей (N2/N3/N4, GTP-U) до безопасности (генерация ключей, программирование SIM-карт через pySIM) и end-to-end тестирования UE.

В рамках учебных и дипломных проектов спроектировал и реализовал отказоустойчивую инфраструктуру в Yandex Cloud, включая Kubernetes-кластер, CI/CD, мониторинг и централизованный сбор логов.  
Регулярно работаю с логами, метриками и диагностикой инцидентов в распределённых системах.

Основной фокус это проектирование и построение отказоустойчивой облачной инфраструктуры, автоматизация процессов и обеспечение надёжности платформ.

- Kubernetes
- Helm

- GitHub

### Курсовой проект. Отказоустойчивая инфраструктура в Yandex Cloud
- [Netology DevOps Course](https://github.com/IlyaBridge/homework/tree/main/netology-devops-course-project)

Описание:
Разработка отказоустойчивой инфраструктуры для веб-приложения в Yandex Cloud с мониторингом, централизованным сбором логов и резервным копированием.

Реализовано:
- Развёртывание инфраструктуры с помощью Terraform и Ansible
- Размещение веб-серверов в разных зонах доступности
- Настройка Application Load Balancer
- Реализация мониторинга с использованием Zabbix
- Централизованный сбор логов через Elasticsearch и Kibana
- Настройка VPC, публичных и приватных подсетей
- Реализация bastion host для безопасного доступа
- Настройка snapshot-бэкапов дисков ВМ

Использованные технологии:
- Yandex Cloud (Compute Cloud, VPC, Application Load Balancer)
- Terraform
- Ansible
- Linux
- Nginx
- Zabbix (monitoring)
- Elasticsearch + Kibana (logging)
- Filebeat
- Networking (VPC, security groups, bastion host)
- Backup и snapshots
